## 1. 根本原因分析

经全面审查，**"不能出字"是以下两个问题叠加导致**：

### 🔴 P0 — INPUT 结构体定义错误（主因）

**文件**: `wubi_ime/main.py` 第 204–235 行  
**严重程度**: 🔴 严重 — 直接导致文字无法输出到目标程序

`_send_output` 方法中使用 ctypes 定义的 `INPUT` 结构体**错误**。Windows API 的 `INPUT` 结构体是一个 **union**（联合体），包含 `type` 字段和联合体 `u`（可以是 `MOUSEINPUT`、`KEYBDINPUT` 或 `HARDWAREINPUT`）。当前代码将其简化为：

```python
# ❌ 当前：错误的扁平结构
class INPUT(ctypes.Structure):
    _fields_ = [
        ("type", ctypes.wintypes.DWORD),      # 4 bytes
        ("ki", KEYBDINPUT),                    # 20 bytes
    ]
# sizeof ≈ 28 bytes（含对齐填充） ❌ 与 Win32 API 期望的 40 bytes 不匹配
```

**影响**: `SendInput` 接收到的结构体大小与系统预期不符，API 调用**静默失败**（返回 0 但不是异常），导致文字无法输出到目标应用程序。这是"不能出字"的**首要原因**。

### 🔴 P0 — 启动时未切换到英文键盘布局（次因）

**文件**: `wubi_ime/main.py` 第 376–379 行

`start()` 方法激活了引擎但**未切换到英文键盘布局**：

```python
# 当前 start() 结尾
self.engine.activate()       # ✓ 激活引擎
# 缺少: self._switch_to_english_layout()  ← 缺失！
```

而在 `_on_activation_hotkey()` 中有此操作。若用户系统默认输入法为微软拼音、搜狗等 TSF 输入法，按键可能被 TSF 先行捕获，键盘钩子根本收不到事件。

**影响**: 在中文 Windows 系统上，按键被系统输入法吞掉，输入法"不出字"。

## 2. 修复方案

### 修复一：修正 INPUT 结构体定义 ✅ 最高优先级

将 `_send_output` 方法中的结构体定义修改为标准 Win32 API 形式：

#### 方案 A（推荐）：使用 `win32con` + `win32api`（更简洁）

```python
import win32con
import win32api
import win32gui

def _send_output(self, text: str):
    """使用 win32api 发送输出"""
    if not text:
        return
    self.keyboard.unhook()
    try:
        for ch in text:
            win32api.keybd_event(0, ord(ch), win32con.KEYEVENTF_UNICODE, 0)
            win32api.keybd_event(0, ord(ch), win32con.KEYEVENTF_UNICODE | win32con.KEYEVENTF_KEYUP, 0)
    except Exception as e:
        print(f"发送输出失败: {e}")
    finally:
        self.keyboard.rehook()
```

#### 方案 B：正确定义 ctypes union 结构

```python
# 定义 INPUT 的联合体
class _INPUT_UNION(ctypes.Union):
    _fields_ = [
        ("mi", ctypes.c_byte * 28),   # MOUSEINPUT padding
        ("ki", KEYBDINPUT),
        ("hi", ctypes.c_byte * 8),    # HARDWAREINPUT padding
    ]

class INPUT(ctypes.Structure):
    _fields_ = [
        ("type", ctypes.wintypes.DWORD),
        ("u", _INPUT_UNION),
    ]

def _send_output(self, text: str):
    if not text:
        return
    self.keyboard.unhook()
    try:
        user32 = ctypes.windll.user32
        inputs = []
        for ch in text:
            # INPUT_KEYBOARD = 1, KEYEVENTF_UNICODE = 0x0004
            ki = KEYBDINPUT(0, ord(ch), 0x0004, 0, 0)
            inp_type = 1  # INPUT_KEYBOARD
            inp = INPUT(inp_type, _INPUT_UNION(ki=ki))
            inputs.append(inp)

            ki_up = KEYBDINPUT(0, ord(ch), 0x0004 | 0x0002, 0, 0)
            inp_up = INPUT(inp_type, _INPUT_UNION(ki=ki_up))
            inputs.append(inp_up)

        if inputs:
            n = len(inputs)
            arr = (INPUT * n)(*inputs)
            ret = user32.SendInput(n, ctypes.byref(arr), ctypes.sizeof(INPUT))
            if ret == 0:
                error_code = ctypes.get_last_error()
                print(f"SendInput 返回 0，错误码: {error_code}")
    except Exception as e:
        print(f"发送输出失败: {e}")
    finally:
        self.keyboard.rehook()
```

    # 不再加锁，由调用方确保

## 4. 性能与架构问题

### 4.1 编码表查询性能

`wubi_table.get_candidates_by_prefix()` 的时间复杂度为 **O(n)**，其中 n = 132,158 条记录。每次按键都会触发一次前缀查询。

**建议方案**：构建 Trie 前缀树索引

```python
class WubiTable:
    def __init__(self, ...):
        ...
        self._trie = {}  # nested dict
    
    def _build_trie(self):
        """构建前缀树"""
        for char, codes in self.char_to_code.items():
            if isinstance(codes, str):
                codes = [codes]
            for code in codes:
                node = self._trie
                for ch in code:
                    if ch not in node:
                        node[ch] = {}
                    node = node[ch]
                if '_chars' not in node:
                    node['_chars'] = []
                if char not in node['_chars']:
                    node['_chars'].append(char)
    
    def get_candidates_by_prefix(self, prefix: str) -> List[str]:
        """前缀查询 O(m)，m = prefix length"""
        node = self._trie
        for ch in prefix:
            if ch not in node:
                return []
            node = node[ch]
        # BFS/DFS collect all chars under this node
        result = []
        stack = [node]
        while stack:
            n = stack.pop()
            if '_chars' in n:
                result.extend(n['_chars'])
            for k, v in n.items():
                if k != '_chars':
                    stack.append(v)
        return result
```

### 4.2 主循环 CPU 占用

当前主循环使用 5ms sleep + 非阻塞队列轮询：

```python
while self._running:
    while True:
        key = self.keyboard.get_key_event(block=False)  # 非阻塞
        if key is None:
            break
        self._on_key(key)
    self._root.update()
    time.sleep(0.005)  # 5ms sleep
```

**建议方案**：改用阻塞读取 + 自适应超时

```python
while self._running:
    key = self.keyboard.get_key_event(block=True, timeout=0.02)
    if key:
        self._on_key(key)
    try:
        self._root.update()
    except tk.TclError:
        break
```

这样在无按键时线程挂起等待，CPU 占用趋近于 0%。
